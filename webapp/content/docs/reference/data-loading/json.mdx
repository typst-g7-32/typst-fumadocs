## `json`

Reads structured data from a JSON file.

The file must contain a valid JSON value, such as object or array. The JSON values will be converted into corresponding Typst values as listed in the [table below](#conversion).

The function returns a dictionary, an array or, depending on the JSON file, another JSON data type.

The JSON files in the example contain objects with the keys `temperature{:typst}`, `unit{:typst}`, and `weather{:typst}`.

## Conversion details

| JSON value | Converted into Typst |
| --- | --- |
| `null{:typst}` | `none{:typst}` |
| bool | [`bool{:typst}`](/docs/reference/foundations/bool/) |
| number | [`float{:typst}`](/docs/reference/foundations/float/) or [`int{:typst}`](/docs/reference/foundations/int/) |
| string | [`str{:typst}`](/docs/reference/foundations/str/) |
| array | [`array{:typst}`](/docs/reference/foundations/array/) |
| object | [`dictionary{:typst}`](/docs/reference/foundations/dictionary/) |

| Typst value | Converted into JSON |
| --- | --- |
| types that can be converted from JSON | corresponding JSON value |
| [`bytes{:typst}`](/docs/reference/foundations/bytes/) | string via [`repr{:typst}`](/docs/reference/foundations/repr/) |
| [`symbol{:typst}`](/docs/reference/foundations/symbol/) | string |
| [`content{:typst}`](/docs/reference/foundations/content/) | an object describing the content |
| other types ([`length{:typst}`](/docs/reference/layout/length/), etc.) | string via [`repr{:typst}`](/docs/reference/foundations/repr/) |

### Notes

- In most cases, JSON numbers will be converted to floats or integers depending on whether they are whole numbers. However, be aware that integers larger than 263-1 or smaller than -263 will be converted to floating-point numbers, which may result in an approximative value.
- Bytes are not encoded as JSON arrays for performance and readability reasons. Consider using [`cbor.encode{:typst}`](/docs/reference/data-loading/cbor/#definitions-encode) for binary data.
- The `repr{:typst}` function is [for debugging purposes only](/docs/reference/foundations/repr/#debugging-only), and its output is not guaranteed to be stable across Typst versions.
