Reads structured data from a YAML file.

The file must contain a valid YAML object or array. The YAML values will be converted into corresponding Typst values as listed in the [table below](#conversion).

The function returns a dictionary, an array or, depending on the YAML file, another YAML data type.

The YAML files in the example contain objects with authors as keys, each with a sequence of their own submapping with the keys "title" and "published".

## Example

<TypstPreview
  code={`
  #let bookshelf(contents) = {
    for (author, works) in contents {
      author
      for work in works [
        - #work.title (#work.published)
      ]
    }
  }

  #bookshelf(
    yaml("scifi-authors.yaml")
  )
`}

## Conversion details

| YAML value | Converted into Typst |
| --- | --- |
| null-values (`null{:typst}`, `~{:typst}` or empty `{:typst}`) | `none{:typst}` |
| boolean | [`bool{:typst}`](/docs/reference/foundations/bool/) |
| number | [`float{:typst}`](/docs/reference/foundations/float/) or [`int{:typst}`](/docs/reference/foundations/int/) |
| string | [`str{:typst}`](/docs/reference/foundations/str/) |
| sequence | [`array{:typst}`](/docs/reference/foundations/array/) |
| mapping | [`dictionary{:typst}`](/docs/reference/foundations/dictionary/) |

| Typst value | Converted into YAML |
| --- | --- |
| types that can be converted from YAML | corresponding YAML value |
| [`bytes{:typst}`](/docs/reference/foundations/bytes/) | string via [`repr{:typst}`](/docs/reference/foundations/repr/) |
| [`symbol{:typst}`](/docs/reference/foundations/symbol/) | string |
| [`content{:typst}`](/docs/reference/foundations/content/) | a mapping describing the content |
| other types ([`length{:typst}`](/docs/reference/layout/length/), etc.) | string via [`repr{:typst}`](/docs/reference/foundations/repr/) |

### Notes

- In most cases, YAML numbers will be converted to floats or integers depending on whether they are whole numbers. However, be aware that integers larger than 263-1 or smaller than -263 will be converted to floating-point numbers, which may result in an approximative value.
- Custom YAML tags are ignored, though the loaded value will still be present.
- Bytes are not encoded as YAML sequences for performance and readability reasons. Consider using [`cbor.encode{:typst}`](/docs/reference/data-loading/cbor/#definitions-encode) for binary data.
- The `repr{:typst}` function is [for debugging purposes only](/docs/reference/foundations/repr/#debugging-only), and its output is not guaranteed to be stable across Typst versions.
