A figure with an optional caption.

Automatically detects its kind to select the correct counting track. For example, figures containing images will be numbered separately from figures containing tables.

## Examples

The example below shows a basic figure with an image:

<TypstPreview
  code={`
@glacier shows a glacier. Glaciers
are complex systems.

#figure(
  image("glacier.jpg", width: 80%),
  caption: [A curious figure.],
) <glacier>
`}
  image='/assets/b9dc3c275cd6f020dfa180755d3cdd2e.png'
  alt='Preview'
/>

You can also insert [tables](/docs/reference/model/table/) into figures to give them a caption. The figure will detect this and automatically use a separate counter.

<TypstPreview
  code={`
#figure(
  table(
    columns: 4,
    [t], [1], [2], [3],
    [y], [0.3s], [0.4s], [0.8s],
  ),
  caption: [Timing results],
)
`}
  image='/assets/fd168e26293d3b952840ef04aba38cf6.png'
  alt='Preview'
/>

This behaviour can be overridden by explicitly specifying the figure's `kind{:typst}`. All figures of the same kind share a common counter.

## Figure behaviour

By default, figures are placed within the flow of content. To make them float to the top or bottom of the page, you can use the [`placement{:typst}`](/docs/reference/model/figure/#parameters-placement) argument.

If your figure is too large and its contents are breakable across pages (e.g. if it contains a large table), then you can make the figure itself breakable across pages as well with this show rule:

```typst
#show figure: set block(breakable: true)
```

See the [block](/docs/reference/layout/block/#parameters-breakable) documentation for more information about breakable and non-breakable blocks.

## Caption customization

You can modify the appearance of the figure's caption with its associated [`caption{:typst}`](/docs/reference/model/figure/#definitions-caption) function. In the example below, we emphasize all captions:

<TypstPreview
  code={`
#show figure.caption: emph

#figure(
  rect[Hello],
  caption: [I am emphasized!],
)
`}
  image='/assets/fd76214814edd5d9a3611f40e27fda0a.png'
  alt='Preview'
/>

By using a [`where{:typst}`](/docs/reference/foundations/function/#definitions-where) selector, we can scope such rules to specific kinds of figures. For example, to position the caption above tables, but keep it below for all other kinds of figures, we could write the following show-set rule:

<TypstPreview
  code={`
#show figure.where(
  kind: table
): set figure.caption(position: top)

#figure(
  table(columns: 2)[A][B][C][D],
  caption: [I'm up here],
)
`}
  image='/assets/e455d8faf40a203e10d4562c478231f4.png'
  alt='Preview'
/>

## Accessibility

You can use the [`alt{:typst}`](/docs/reference/model/figure/#parameters-alt) parameter to provide an [alternative description](/docs/guides/accessibility/#textual-representations) of the figure for screen readers and other Assistive Technology (AT). Refer to [its documentation](/docs/reference/model/figure/#parameters-alt) to learn more.

You can use figures to add alternative descriptions to paths, shapes, or visualizations that do not have their own `alt{:typst}` parameter. If your graphic is purely decorative and does not have a semantic meaning, consider wrapping it in [`pdf.artifact{:typst}`](/docs/reference/pdf/artifact/) instead, which will hide it from AT when exporting to PDF.

AT will always read the figure at the point where it appears in the document, regardless of its [`placement{:typst}`](/docs/reference/model/figure/#parameters-placement). Put its markup where it would make the most sense in the reading order.

```typst
#figure(
  body,
  alt: none | str,
  placement: none | auto | alignment,
  scope: str,
  caption: none | content,
  kind: auto | str | function,
  supplement: none | auto | content | function,
  numbering: none | str | function,
  gap: length,
  outlined: bool
) -> content
```

### Definitions
The caption of a figure. This element can be used in set and show rules to customize the appearance of captions for all figures or figures of a specific kind.

In addition to its `position{:typst}` and `body{:typst}`, the `caption{:typst}` also provides the figure's `kind{:typst}`, `supplement{:typst}`, `counter{:typst}`, and `numbering{:typst}` as fields. These parts can be used in [`where{:typst}`](/docs/reference/foundations/function/#definitions-where) selectors and show rules to build a completely custom caption.

<TypstPreview
  code={`
#show figure.caption: emph

#figure(
  rect[Hello],
  caption: [A rectangle],
)
`}
  image='/assets/ffd45a7b793ed7875c6f4d1b596e1c88.png'
  alt='Preview'
/>

```typst
#figure.caption(
  position: alignment,
  separator: auto | content,
  body
) -> content
```
