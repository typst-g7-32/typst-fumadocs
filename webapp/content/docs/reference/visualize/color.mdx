A color in a specific color space.

Typst supports:

- sRGB through the [`rgb{:typst}` function](/docs/reference/visualize/color/#definitions-rgb)
- Device CMYK through the [`cmyk{:typst}` function](/docs/reference/visualize/color/#definitions-cmyk)
- D65 Gray through the [`luma{:typst}` function](/docs/reference/visualize/color/#definitions-luma)
- Oklab through the [`oklab{:typst}` function](/docs/reference/visualize/color/#definitions-oklab)
- Oklch through the [`oklch{:typst}` function](/docs/reference/visualize/color/#definitions-oklch)
- Linear RGB through the [`color.linear-rgb{:typst}` function](/docs/reference/visualize/color/#definitions-linear-rgb)
- HSL through the [`color.hsl{:typst}` function](/docs/reference/visualize/color/#definitions-hsl)
- HSV through the [`color.hsv{:typst}` function](/docs/reference/visualize/color/#definitions-hsv)

## Example

```typst
#rect(fill: aqua)
```

## Predefined color maps

Typst also includes a number of preset color maps that can be used for [gradients](/docs/reference/visualize/gradient/#stops). These are simply arrays of colors defined in the module `color.map{:typst}`.

```typst
#circle(fill: gradient.linear(..color.map.crest))
```

| Map | Details |
| --- | --- |
| `turbo{:typst}` | A perceptually uniform rainbow-like color map. Read [this blog post](https://ai.googleblog.com/2019/08/turbo-improved-rainbow-colormap-for.html) for more details. |
| `cividis{:typst}` | A blue to gray to yellow color map. See [this blog post](https://bids.github.io/colormap/) for more details. |
| `rainbow{:typst}` | Cycles through the full color spectrum. This color map is best used by setting the interpolation color space to [HSL](/docs/reference/visualize/color/#definitions-hsl). The rainbow gradient is **not suitable** for data visualization because it is not perceptually uniform, so the differences between values become unclear to your readers. It should only be used for decorative purposes. |
| `spectral{:typst}` | Red to yellow to blue color map. |
| `viridis{:typst}` | A purple to teal to yellow color map. |
| `inferno{:typst}` | A black to red to yellow color map. |
| `magma{:typst}` | A black to purple to yellow color map. |
| `plasma{:typst}` | A purple to pink to yellow color map. |
| `rocket{:typst}` | A black to red to white color map. |
| `mako{:typst}` | A black to teal to white color map. |
| `coolwarm{:typst}` | A blue to white to red color map with smooth transitions. |
| `vlag{:typst}` | A light blue to white to red color map. |
| `icefire{:typst}` | A light teal to black to orange color map. |
| `flare{:typst}` | A orange to purple color map that is perceptually uniform. |
| `crest{:typst}` | A light green to blue color map. |

Some popular presets are not included because they are not available under a free licence. Others, like [Jet](https://jakevdp.github.io/blog/2014/10/16/how-bad-is-your-colormap/), are not included because they are not color blind friendly. Feel free to use or create a package with other presets that are useful to you!

### `luma`

Create a grayscale color.

A grayscale color is represented internally by a single `lightness{:typst}` component.

These components are also available using the [`components{:typst}`](/docs/reference/visualize/color/#definitions-components) method.

```typst
#for x in range(250, step: 50) {
  box(square(fill: luma(x)))
}
```

### `oklab`

Create an [Oklab](https://bottosson.github.io/posts/oklab/) color.

This color space is well suited for the following use cases:

- Color manipulation such as saturating while keeping perceived hue
- Creating grayscale images with uniform perceived lightness
- Creating smooth and uniform color transition and gradients

A linear Oklab color is represented internally by an array of four components:

- lightness ([`ratio{:typst}`](/docs/reference/layout/ratio/))
- a ([`float{:typst}`](/docs/reference/foundations/float/) or [`ratio{:typst}`](/docs/reference/layout/ratio/). Ratios are relative to `0.4{:typst}`; meaning `50%{:typst}` is equal to `0.2{:typst}`)
- b ([`float{:typst}`](/docs/reference/foundations/float/) or [`ratio{:typst}`](/docs/reference/layout/ratio/). Ratios are relative to `0.4{:typst}`; meaning `50%{:typst}` is equal to `0.2{:typst}`)
- alpha ([`ratio{:typst}`](/docs/reference/layout/ratio/))

These components are also available using the [`components{:typst}`](/docs/reference/visualize/color/#definitions-components) method.

```typst
#square(
  fill: oklab(27%, 20%, -3%, 50%)
)
```

### `linear-rgb`

Create an RGB(A) color with linear luma.

This color space is similar to sRGB, but with the distinction that the color component are not gamma corrected. This makes it easier to perform color operations such as blending and interpolation. Although, you should prefer to use the [`oklab{:typst}` function](/docs/reference/visualize/color/#definitions-oklab) for these.

A linear RGB(A) color is represented internally by an array of four components:

- red ([`ratio{:typst}`](/docs/reference/layout/ratio/))
- green ([`ratio{:typst}`](/docs/reference/layout/ratio/))
- blue ([`ratio{:typst}`](/docs/reference/layout/ratio/))
- alpha ([`ratio{:typst}`](/docs/reference/layout/ratio/))

These components are also available using the [`components{:typst}`](/docs/reference/visualize/color/#definitions-components) method.

```typst
#square(fill: color.linear-rgb(
```

### `components`

Extracts the components of this color.

The size and values of this array depends on the color space. You can obtain the color space using [`space{:typst}`](/docs/reference/visualize/color/#definitions-space). Below is a table of the color spaces and their components:

| Color space | C1 | C2 | C3 | C4 |
| --- | --- | --- | --- | --- |
| [`luma{:typst}`](/docs/reference/visualize/color/#definitions-luma) | Lightness |  |  |  |
| [`oklab{:typst}`](/docs/reference/visualize/color/#definitions-oklab) | Lightness | `a{:typst}` | `b{:typst}` | Alpha |
| [`oklch{:typst}`](/docs/reference/visualize/color/#definitions-oklch) | Lightness | Chroma | Hue | Alpha |
| [`linear-rgb{:typst}`](/docs/reference/visualize/color/#definitions-linear-rgb) | Red | Green | Blue | Alpha |
| [`rgb{:typst}`](/docs/reference/visualize/color/#definitions-rgb) | Red | Green | Blue | Alpha |
| [`cmyk{:typst}`](/docs/reference/visualize/color/#definitions-cmyk) | Cyan | Magenta | Yellow | Key |
| [`hsl{:typst}`](/docs/reference/visualize/color/#definitions-hsl) | Hue | Saturation | Lightness | Alpha |
| [`hsv{:typst}`](/docs/reference/visualize/color/#definitions-hsv) | Hue | Saturation | Value | Alpha |

For the meaning and type of each individual value, see the documentation of the corresponding color space. The alpha component is optional and only included if the `alpha{:typst}` argument is `true{:typst}`. The length of the returned array depends on the number of components and whether the alpha component is included.

```typst
// note that the alpha component is included by default
#rgb(40%, 60%, 80%).components()
```

### `mix`

In color spaces with a hue component (hsl, hsv, oklch), only two colors can be mixed at once. Mixing more than two colors in such a space will result in an error!

```typst

#block(fill: red.mix(blue))
#block(fill: red.mix(blue, space: rgb))
#block(fill: color.mix(red, blue, white))
#block(fill: color.mix((red, 70%), (blue, 30%)))
```

### `transparentize`

Makes a color more transparent by a given factor.

This method is relative to the existing alpha value. If the scale is positive, calculates `alpha - alpha * scale{:typst}`. Negative scales behave like `color.opacify(-scale){:typst}`.

```typst
#block(fill: red)[opaque]
#block(fill: red.transparentize(50%))[half red]
#block(fill: red.transparentize(75%))[quarter red]
```

### `opacify`

Makes a color more opaque by a given scale.

This method is relative to the existing alpha value. If the scale is positive, calculates `alpha + scale - alpha * scale{:typst}`. Negative scales behave like `color.transparentize(-scale){:typst}`.

```typst
#let half-red = red.transparentize(50%)
#block(fill: half-red.opacify(100%))[opaque]
#block(fill: half-red.opacify(50%))[three quarters red]
#block(fill: half-red.opacify(-50%))[one quarter red]
```
