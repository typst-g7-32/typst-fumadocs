Represents a date, a time, or a combination of both.

Can be created by either specifying a custom datetime using this type's constructor function or getting the current date with [`datetime.today{:typst}`](/docs/reference/foundations/datetime/#definitions-today).

## Datetime and Duration

You can get a [duration](/docs/reference/foundations/duration/) by subtracting two datetime:

<TypstPreview
  code={`
  #let first-of-march = datetime(day: 1, month: 3, year: 2024)
  #let first-of-jan = datetime(day: 1, month: 1, year: 2024)
  #let distance = first-of-march - first-of-jan
  #distance.hours()
`}
  image='/assets/c4920f9ef579222c3ca2c76488051bfc.png'
  alt='Preview'
  editable={true}
/>

You can also add/subtract a datetime and a duration to retrieve a new, offset datetime:

<TypstPreview
  code={`

  #two-days-earlier.display() \
  #two-days-later.display()

## Format

You can specify a customized formatting using the [`display{:typst}`](/docs/reference/foundations/datetime/#definitions-display) method. The format of a datetime is specified by providing _components_ with a specified number of _modifiers_. A component represents a certain part of the datetime that you want to display, and with the help of modifiers you can define how you want to display that component. In order to display a component, you wrap the name of the component in square brackets (e.g. `[year]{:typst}` will display the year). In order to add modifiers, you add a space after the component name followed by the name of the modifier, a colon and the value of the modifier (e.g. `[month repr:short]{:typst}` will display the short representation of the month).

The possible combination of components and their respective modifiers is as follows:

- `year{:typst}`: Displays the year of the datetime. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the year is padded.
  - `repr{:typst}` Can be either `full{:typst}` in which case the full year is displayed or `last_two{:typst}` in which case only the last two digits are displayed.
  - `sign{:typst}`: Can be either `automatic{:typst}` or `mandatory{:typst}`. Specifies when the sign should be displayed.
- `month{:typst}`: Displays the month of the datetime. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the month is padded.
  - `repr{:typst}`: Can be either `numerical{:typst}`, `long{:typst}` or `short{:typst}`. Specifies if the month should be displayed as a number or a word. Unfortunately, when choosing the word representation, it can currently only display the English version. In the future, it is planned to support localization.
- `day{:typst}`: Displays the day of the datetime. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the day is padded.
- `week_number{:typst}`: Displays the week number of the datetime. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the week number is padded.
  - `repr{:typst}`: Can be either `ISO{:typst}`, `sunday{:typst}` or `monday{:typst}`. In the case of `ISO{:typst}`, week numbers are between 1 and 53, while the other ones are between 0 and 53.
- `weekday{:typst}`: Displays the weekday of the date. 
  - `repr{:typst}` Can be either `long{:typst}`, `short{:typst}`, `sunday{:typst}` or `monday{:typst}`. In the case of `long{:typst}` and `short{:typst}`, the corresponding English name will be displayed (same as for the month, other languages are currently not supported). In the case of `sunday{:typst}` and `monday{:typst}`, the numerical value will be displayed (assuming Sunday and Monday as the first day of the week, respectively).
  - `one_indexed{:typst}`: Can be either `true{:typst}` or `false{:typst}`. Defines whether the numerical representation of the week starts with 0 or 1.
- `hour{:typst}`: Displays the hour of the date. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the hour is padded.
  - `repr{:typst}`: Can be either `24{:typst}` or `12{:typst}`. Changes whether the hour is displayed in the 24-hour or 12-hour format.
- `period{:typst}`: The AM/PM part of the hour 
  - `case{:typst}`: Can be `lower{:typst}` to display it in lower case and `upper{:typst}` to display it in upper case.
- `minute{:typst}`: Displays the minute of the date. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the minute is padded.
- `second{:typst}`: Displays the second of the date. 
  - `padding{:typst}`: Can be either `zero{:typst}`, `space{:typst}` or `none{:typst}`. Specifies how the second is padded.

Keep in mind that not always all components can be used. For example, if you create a new datetime with `datetime(year: 2023, month: 10, day: 13){:typst}`, it will be stored as a plain date internally, meaning that you cannot use components such as `hour{:typst}` or `minute{:typst}`, which would only work on datetimes that have a specified time.

## Constructor
Creates a new datetime.

You can specify the [datetime](/docs/reference/foundations/datetime/) using a year, month, day, hour, minute, and second.

_Note_: Depending on which components of the datetime you specify, Typst will store it in one of the following three ways:

- If you specify year, month and day, Typst will store just a date.
- If you specify hour, minute and second, Typst will store just a time.
- If you specify all of year, month, day, hour, minute and second, Typst will store a full datetime.

Depending on how it is stored, the [`display{:typst}`](/docs/reference/foundations/datetime/#definitions-display) method will choose a different formatting by default.

## Methods
Returns the current date.

In the CLI, this can be overridden with the `--creation-timestamp{:typst}` argument or by setting the [`SOURCE_DATE_EPOCH{:typst}`](https://reproducible-builds.org/specs/source-date-epoch/) environment variable. In both cases, the value should be given as a UNIX timestamp.

Displays the datetime in a specified format.

Depending on whether you have defined just a date, a time or both, the default format will be different. If you specified a date, it will be `[year]-[month]-[day]{:typst}`. If you specified a time, it will be `[hour]:[minute]:[second]{:typst}`. In the case of a datetime, it will be `[year]-[month]-[day] [hour]:[minute]:[second]{:typst}`.

See the [format syntax](/docs/reference/foundations/datetime/#format) for more information.
