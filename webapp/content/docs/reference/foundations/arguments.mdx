Captured arguments to a function.

## Argument Sinks

Like built-in functions, custom functions can also take a variable number of arguments. You can specify an _argument sink_ which collects all excess arguments as `..sink{:typst}`. The resulting `sink{:typst}` value is of the `arguments{:typst}` type. It exposes methods to access the positional and named arguments.

<TypstPreview
  code={`
  #let format(title, ..authors) = {
    let by = authors
      .pos()
      .join(", ", last: " and ")

    [*#title* \ _Written by #by;_]
  }

  #format("ArtosFlow", "Jane", "Joe")
`}
  image='/assets/d6ce7ebd806b82775abf2d566f8c412.png'
  alt='Preview'
  editable={true}
/>

## Spreading

Inversely to an argument sink, you can _spread_ arguments, arrays and dictionaries into a function call with the `..spread{:typst}` operator:

<TypstPreview
  code={`
  #let array = (2, 3, 5)
  #calc.min(..array)
  #let dict = (fill: blue)
  #text(..dict)[Hello]
`}
  image='/assets/91c9aab47f6ac6ae8183c670c0a9cc09.png'
  alt='Preview'
  editable={true}
/>

## Constructor
Construct spreadable arguments in place.

This function behaves like `let args(..sink) = sink{:typst}`.

<TypstPreview
  code={`

  #box(..args)

## Methods
The number of arguments, positional or named.

Returns the positional argument at the specified index, or the named argument with the specified name.

If the key is an [integer](/docs/reference/foundations/int/), this is equivalent to first calling [`pos{:typst}`](/docs/reference/foundations/arguments/#definitions-pos) and then [`array.at{:typst}`](/docs/reference/foundations/array/#definitions-at). If it is a [string](/docs/reference/foundations/str/), this is equivalent to first calling [`named{:typst}`](/docs/reference/foundations/arguments/#definitions-named) and then [`dictionary.at{:typst}`](/docs/reference/foundations/dictionary/#definitions-at).

```typst
#arguments.at(
  key,
  default: any
) -> any
```
