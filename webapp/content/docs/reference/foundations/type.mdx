Describes a kind of value.

To style your document, you need to work with values of different kinds: Lengths specifying the size of your elements, colors for your text and shapes, and more. Typst categorizes these into clearly defined _types_ and tells you where it expects which type of value.

Apart from basic types for numeric values and [typical](/docs/reference/foundations/int/) [types](/docs/reference/foundations/float/) [known](/docs/reference/foundations/str/) [from](/docs/reference/foundations/array/) [programming](/docs/reference/foundations/dictionary/) languages, Typst provides a special type for [_content._](/docs/reference/foundations/content/) A value of this type can hold anything that you can enter into your document: Text, elements like headings and shapes, and style information.

## Example

<TypstPreview
  code={`
  #let x = 10
  #if type(x) == int [
    #x is an integer!
  ] else [
    #x is another value...
  ]

  An image is of type
  #type(image("glacier.jpg")).
`}
  image='/typst-fumadocs/assets/7538c768430ee75e747b4f97560d4ecf'
  alt='Preview'
  editable={false}
/>

The type of `10{:typst}` is `int{:typst}`. Now, what is the type of `int{:typst}` or even `type{:typst}`?

<TypstPreview
  code={`
  #type(int) \
  #type(type)
`}
  image='/typst-fumadocs/assets/1ea220672ff0a816e76e846567e22fe0'
  alt='Preview'
  editable={true}
/>

Unlike other types like `int{:typst}`, [none](/docs/reference/foundations/none/) and [auto](/docs/reference/foundations/auto/) do not have a name representing them. To test if a value is one of these, compare your value to them directly, e.g:

<TypstPreview
  code={`
  #let val = none
  #if val == none [
    Yep, it's none.
  ]
`}
  image='/typst-fumadocs/assets/9fe64b0d6307e76aa133e5f4f468d494'
  alt='Preview'
  editable={true}
/>

Note that `type{:typst}` will return [`content{:typst}`](/docs/reference/foundations/content/) for all document elements. To programmatically determine which kind of content you are dealing with, see [`content.func{:typst}`](/docs/reference/foundations/content/#definitions-func).

## Constructor
Determines a value's type.

<TypstPreview
  code={`
  #type(12) \
  #type(14.7) \
  #type("hello") \
  #type(<glacier>) \
  #type([Hi]) \
  #type(x => x + 1) \
  #type(type)
`}
  image='/typst-fumadocs/assets/3bff018780f2b4261ae9dc20c2e887a'
  alt='Preview'
  editable={true}
/>

```typst
#type(
  value
) -> type
```
