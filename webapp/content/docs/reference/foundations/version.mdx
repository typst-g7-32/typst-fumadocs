A version with an arbitrary number of components.

The first three components have names that can be used as fields: `major{:typst}`, `minor{:typst}`, `patch{:typst}`. All following components do not have names.

The list of components is semantically extended by an infinite list of zeros. This means that, for example, `0.8{:typst}` is the same as `0.8.0{:typst}`. As a special case, the empty version (that has no components at all) is the same as `0{:typst}`, `0.0{:typst}`, `0.0.0{:typst}`, and so on.

The current version of the Typst compiler is available as `sys.version{:typst}`.

You can convert a version to an array of explicitly given components using the [`array{:typst}`](/docs/reference/foundations/array/) constructor.

## Constructor
Creates a new version.

It can have any number of components (even zero).

<TypstPreview
  code={`
#version() 
#version(1) 
#version(1, 2, 3, 4) 
#version((1, 2, 3, 4)) 
#version((1, 2), 3)
`}
  image='/assets/171d7fe9db3c91b277e567ab934a88a9.png'
  alt='Preview'
/>

As a practical use case, this allows comparing the current version ([`sys.version{:typst}`](/docs/reference/foundations/version/)) to a specific one.

<TypstPreview
  code={`
Current version: #sys.version 
#(sys.version >= version(0, 14, 0)) 
#(version(3, 2, 0) > version(4, 1, 0))
`}
  image='/assets/9abdaee6d68422e95c9d5c76afd7cdcf.png'
  alt='Preview'
/>

## Methods
Retrieves a component of a version.

The returned integer is always non-negative. Returns `0{:typst}` if the version isn't specified to the necessary length.

```typst
#version.at(
  index
) -> int
```
