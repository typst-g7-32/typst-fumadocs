A sequence of values.

You can construct an array by enclosing a comma-separated sequence of values in parentheses. The values do not have to be of the same type.

You can access and update array items with the `.at(){:typst}` method. Indices are zero-based and negative indices wrap around to the end of the array. You can iterate over an array using a [for loop](/docs/reference/scripting/#loops). Arrays can be added together with the `+{:typst}` operator, [joined together](/docs/reference/scripting/#blocks) and multiplied with integers.

**Note:** An array of length one needs a trailing comma, as in `(1,){:typst}`. This is to disambiguate from a simple parenthesized expressions like `(1 + 2) * 3{:typst}`. An empty array is written as `(){:typst}`.

Note that this function is only intended for conversion of a collection-like value to an array, not for creation of an array from individual items. Use the array syntax `(1, 2, 3){:typst}` (or `(1,){:typst}` for a single-element array) instead.

Zips the array with other arrays.

Returns an array of arrays, where the `i{:typst}`th inner array contains all the `i{:typst}`th elements from each original array.

If the arrays to be zipped have different lengths, they are zipped up to the last element of the shortest array and all remaining elements are ignored.

This function is variadic, meaning that you can zip multiple arrays together at once: `(1, 2).zip(("A", "B"), (10, 20)){:typst}` yields `((1, "A", 10), (2, "B", 20)){:typst}`.

If the array is empty, returns `none{:typst}`, otherwise, returns the result of the reduction.

For arrays with at least one element, this is the same as [`array.fold{:typst}`](/docs/reference/foundations/array/#definitions-fold) with the first element of the array as the initial accumulator value, folding every subsequent element into it.
