A whole number.

The number can be negative, zero, or positive. As Typst uses 64 bits to store integers, integers cannot be smaller than `-9223372036854775808{:typst}` or larger than `9223372036854775807{:typst}`. Integer literals are always positive, so a negative integer such as `-1{:typst}` is semantically the negation `-{:typst}` of the positive literal `1{:typst}`. A positive integer greater than the maximum value and a negative integer less than or equal to the minimum value cannot be represented as an integer literal, and are instead parsed as a `float{:typst}`. The minimum integer value can still be obtained through integer arithmetic.

The number can also be specified as hexadecimal, octal, or binary by starting it with a zero followed by either `x{:typst}`, `o{:typst}`, or `b{:typst}`.

You can convert a value to an integer with this type's constructor.

## Example

```typst
#(1 + 2) \
#(2 - 5) \
#(3 + 4 < 8)

#0xff \
#0o10 \
#0b1001
```

## Constructor

### `int`

Converts a value to an integer. Raises an error if there is an attempt to parse an invalid string or produce an integer that doesn't fit into a 64-bit signed integer.

- Booleans are converted to `0{:typst}` or `1{:typst}`.
- Floats and decimals are rounded to the next 64-bit integer towards zero.
- Strings are parsed in base 10 by default.

```typst
#int(false) \
#int(true) \
#int(2.7) \
#int(decimal("3.8")) \
#(int("27") + int("4")) \
#(int("beef", base: 16))
```

## Methods

### `signum`

Calculates the sign of an integer.

- If the number is positive, returns `1{:typst}`.
- If the number is negative, returns `-1{:typst}`.
- If the number is zero, returns `0{:typst}`.

```typst
#(5).signum() \
#(-5).signum() \
#(0).signum()
```

```#int.signum() -> int```

### `bit-not`

Calculates the bitwise NOT of an integer.

For the purposes of this function, the operand is treated as a signed integer of 64 bits.

```typst
#4.bit-not() \
#(-1).bit-not()
```

```#int.bit-not() -> int```

### `bit-and`

For the purposes of this function, the operands are treated as signed integers of 64 bits.

### `bit-lshift`

Shifts the operand's bits to the left by the specified amount.

For the purposes of this function, the operand is treated as a signed integer of 64 bits. An error will occur if the result is too large to fit in a 64-bit integer.

```typst
#33.bit-lshift(2) \
#(-1).bit-lshift(3)
```

### `bit-rshift`

Shifts the operand's bits to the right by the specified amount. Performs an arithmetic shift by default (extends the sign bit to the left, such that negative numbers stay negative), but that can be changed by the `logical{:typst}` parameter.

For the purposes of this function, the operand is treated as a signed integer of 64 bits.

```typst
#64.bit-rshift(2) \
#(-8).bit-rshift(2) \
#(-8).bit-rshift(2, logical: true)
```
