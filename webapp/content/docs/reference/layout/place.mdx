Places content relatively to its parent container.

Placed content can be either overlaid (the default) or floating. Overlaid content is aligned with the parent container according to the given [`alignment{:typst}`](/docs/reference/layout/place/#parameters-alignment), and shown over any other content added so far in the container. Floating content is placed at the top or bottom of the container, displacing other content down or up respectively. In both cases, the content position can be adjusted with [`dx{:typst}`](/docs/reference/layout/place/#parameters-dx) and [`dy{:typst}`](/docs/reference/layout/place/#parameters-dy) offsets without affecting the layout.

The parent can be any container such as a [`block{:typst}`](/docs/reference/layout/block/), [`box{:typst}`](/docs/reference/layout/box/), [`rect{:typst}`](/docs/reference/visualize/rect/), etc. A top level `place{:typst}` call will place content directly in the text area of the current page. This can be used for absolute positioning on the page: with a `top + left{:typst}` [`alignment{:typst}`](/docs/reference/layout/place/#parameters-alignment), the offsets `dx{:typst}` and `dy{:typst}` will set the position of the element's top left corner relatively to the top left corner of the text area. For absolute positioning on the full page including margins, you can use `place{:typst}` in [`page.foreground{:typst}`](/docs/reference/layout/page/#parameters-foreground) or [`page.background{:typst}`](/docs/reference/layout/page/#parameters-background).

## Effect on the position of other elements

Overlaid elements don't take space in the flow of content, but a `place{:typst}` call inserts an invisible block-level element in the flow. This can affect the layout by breaking the current paragraph. To avoid this, you can wrap the `place{:typst}` call in a [`box{:typst}`](/docs/reference/layout/box/) when the call is made in the middle of a paragraph. The alignment and offsets will then be relative to this zero-size box. To make sure it doesn't interfere with spacing, the box should be attached to a word using a word joiner.

For example, the following defines a function for attaching an annotation to the following word:

<TypstPreview
  code={`
  #let annotate(..args) = {
    box(place(..args))
    sym.wj
    h(0pt, weak: true)
  }

  A placed #annotate(square(), dy: 2pt)
  square in my text.
`}
  image='/assets/40826a3ec000a798ea7be101e1b645d6.png'
  alt='Preview'
  editable={true}
/>

The zero-width weak spacing serves to discard spaces between the function call and the next word.

## Accessibility

Assistive Technology (AT) will always read the placed element at the point where it logically appears in the document, regardless of where this function physically moved it. Put its markup where it would make the most sense in the reading order.

```typst
#place(
  alignment,
  scope: str,
  float: bool,
  clearance: length,
  dx: relative,
  dy: relative,
  body
) -> content
```

### Definitions
Asks the layout algorithm to place pending floating elements before continuing with the content.

This is useful for preventing floating figures from spilling into the next section.

<TypstPreview
  code={`
  #lorem(15)

  #figure(
    rect(width: 100%, height: 50pt),
    placement: auto,
    caption: [A rectangle],
  )

  #place.flush()

  This text appears after the figure.
`}
