Arranges content in a grid.

The grid element allows you to arrange content in a grid. You can define the number of rows and columns, as well as the size of the gutters between them. There are multiple sizing modes for columns and rows that can be used to create complex layouts.

While the grid and table elements work very similarly, they are intended for different use cases and carry different semantics. The grid element is intended for presentational and layout purposes, while the [`table{:typst}`](/docs/reference/model/table/) element is intended for, in broad terms, presenting multiple related data points. Set and show rules on one of these elements do not affect the other. Refer to the [Accessibility Section](/docs/reference/layout/grid/#accessibility) to learn how grids and tables are presented to users of Assistive Technology (AT) like screen readers.

## Sizing the tracks

A grid's sizing is determined by the track sizes specified in the arguments. There are multiple sizing parameters: [`columns{:typst}`](/docs/reference/layout/grid/#parameters-columns), [`rows{:typst}`](/docs/reference/layout/grid/#parameters-rows) and [`gutter{:typst}`](/docs/reference/layout/grid/#parameters-gutter). Because each of the sizing parameters accepts the same values, we will explain them just once, here. Each sizing argument accepts an array of individual track sizes. A track size is either:

- `auto{:typst}`: The track will be sized to fit its contents. It will be at most as large as the remaining space. If there is more than one `auto{:typst}` track width, and together they claim more than the available space, the `auto{:typst}` tracks will fairly distribute the available space among themselves.
- A fixed or relative length (e.g. `10pt{:typst}` or `20% - 1cm{:typst}`): The track will be exactly of this size.
- A fractional length (e.g. `1fr{:typst}`): Once all other tracks have been sized, the remaining space will be divided among the fractional tracks according to their fractions. For example, if there are two fractional tracks, each with a fraction of `1fr{:typst}`, they will each take up half of the remaining space.

To specify a single track, the array can be omitted in favor of a single value. To specify multiple `auto{:typst}` tracks, enter the number of tracks instead of an array. For example, `columns:{:typst}` `3{:typst}` is equivalent to `columns:{:typst}` `(auto, auto, auto){:typst}`.

## Styling the grid

The grid and table elements work similarly. For a hands-on explanation, refer to the [Table Guide](/docs/guides/tables/#fills); for a quick overview, continue reading.

The grid's appearance can be customized through different parameters. These are the most important ones:

- [`align{:typst}`](/docs/reference/layout/grid/#parameters-align) to change how cells are aligned
- [`inset{:typst}`](/docs/reference/layout/grid/#parameters-inset) to optionally add internal padding to cells
- [`fill{:typst}`](/docs/reference/layout/grid/#parameters-fill) to give cells a background
- [`stroke{:typst}`](/docs/reference/layout/grid/#parameters-stroke) to optionally enable grid lines with a certain stroke

To meet different needs, there are various ways to set them.

If you need to override the above options for individual cells, you can use the [`grid.cell{:typst}`](/docs/reference/layout/grid/#definitions-cell) element. Likewise, you can override individual grid lines with the [`grid.hline{:typst}`](/docs/reference/layout/grid/#definitions-hline) and [`grid.vline{:typst}`](/docs/reference/layout/grid/#definitions-vline) elements.

To configure an overall style for a grid, you may instead specify the option in any of the following fashions:

- As a single value that applies to all cells.
- As an array of values corresponding to each column. The array will be cycled if there are more columns than the array has items.
- As a function in the form of `(x, y) => value{:typst}`. It receives the cell's column and row indices (both starting from zero) and should return the value to apply to that cell.

<TypstPreview
  code={`
  #grid(
    columns: 5,

    // By a single value
    align: center,
    // By a single but more complicated value
    inset: (x: 2pt, y: 3pt),
    // By an array of values (cycling)
    fill: (rgb("#239dad50"), none),
    // By a function that returns a value
    stroke: (x, y) => if calc.rem(x + y, 3) == 0 { 0.5pt },

    ..range(5 * 3).map(n => numbering("A", n + 1))
  )
`}
  image='/typst-fumadocs/assets/5ba008cb625404be10f2b8abc3412fab'
  alt='Preview'
  editable={true}
/>

On top of that, you may [apply styling rules](/docs/reference/styling/) to [`grid{:typst}`](/docs/reference/layout/grid/) and [`grid.cell{:typst}`](/docs/reference/layout/grid/#definitions-cell). Especially, the [`x{:typst}`](/docs/reference/layout/grid/#definitions-cell-x) and [`y{:typst}`](/docs/reference/layout/grid/#definitions-cell-y) fields of `grid.cell{:typst}` can be used in a [`where{:typst}`](/docs/reference/foundations/function/#definitions-where) selector, making it possible to style cells at specific columns or rows, or individual positions.

### Stroke styling precedence

As explained above, there are three ways to set the stroke of a grid cell: through [`grid.cell{:typst}`'s `stroke{:typst}` field](/docs/reference/layout/grid/#definitions-cell-stroke), by using [`grid.hline{:typst}`](/docs/reference/layout/grid/#definitions-hline) and [`grid.vline{:typst}`](/docs/reference/layout/grid/#definitions-vline), or by setting the [`grid{:typst}`'s `stroke{:typst}` field](/docs/reference/layout/grid/#parameters-stroke). When multiple of these settings are present and conflict, the `hline{:typst}` and `vline{:typst}` settings take the highest precedence, followed by the `cell{:typst}` settings, and finally the `grid{:typst}` settings.

Furthermore, strokes of a repeated grid header or footer will take precedence over regular cell strokes.

## Accessibility

Grids do not carry any special semantics. Assistive Technology (AT) does not offer the ability to navigate two-dimensionally by cell in grids. If you want to present tabular data, use the [`table{:typst}`](/docs/reference/model/table/) element instead.

AT will read the grid cells in their semantic order. Usually, this is the order in which you passed them to the grid. However, if you manually positioned them using [`grid.cell{:typst}`'s `x{:typst}` and `y{:typst}` arguments](/docs/reference/layout/grid/#definitions-cell-x), cells will be read row by row, from left to right (in left-to-right documents). A cell will be read when its position is first reached.

```typst
#grid(
  columns: auto | int | relative | fraction | array,
  rows: auto | int | relative | fraction | array,
  gutter: auto | int | relative | fraction | array,
  column-gutter: auto | int | relative | fraction | array,
  row-gutter: auto | int | relative | fraction | array,
  inset: relative | array | dictionary | function,
  align: auto | array | alignment | function,
  fill: none | color | gradient | array | tiling | function,
  stroke: none | length | color | gradient | array | stroke | tiling | dictionary | function,
  children
) -> content
```

### Definitions
A cell in the grid. You can use this function in the argument list of a grid to override grid style properties for an individual cell or manually positioning it within the grid. You can also use this function in show rules to apply certain styles to multiple cells at once.

For example, you can override the position and stroke for a single cell:

<TypstPreview
  code={`
  #set text(15pt, font: "Noto Sans Symbols 2")
  #show regex("[♚-♟︎]"): set text(fill: rgb("21212A"))
  #show regex("[♔-♙]"): set text(fill: rgb("111015"))

  #grid(
    fill: (x, y) => rgb(
      if calc.odd(x + y) { "7F8396" }
      else { "EFF0F3" }
    ),
    columns: (1em,) * 8,
    rows: 1em,
    align: center + horizon,

    [♖], [♘], [♗], [♕], [♔], [♗], [♘], [♖],
    [♙], [♙], [♙], [♙], [],  [♙], [♙], [♙],
    grid.cell(
      x: 4, y: 3,
      stroke: blue.transparentize(60%)
    )[♙],

    ..(grid.cell(y: 6)[♟],) * 8,
    ..([♜], [♞], [♝], [♛], [♚], [♝], [♞], [♜])

  )

  stroke: none | length | color | gradient | stroke | tiling | dictionary,
  breakable: auto | bool
) -> content
```

A horizontal line in the grid.

Overrides any per-cell stroke, including stroke specified through the grid's `stroke{:typst}` field. Can cross spacing between cells created through the grid's `column-gutter{:typst}` option.

An example for this function can be found at the [`table.hline{:typst}`](/docs/reference/model/table/#definitions-hline) element.

  y: auto | int,
  start: int,
  end: none | int,
  stroke: none | length | color | gradient | stroke | tiling | dictionary,
  position: alignment
) -> content
```
