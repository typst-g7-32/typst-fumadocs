Moves content without affecting layout.

The `move{:typst}` function allows you to move content while the layout still 'sees' it at the original positions. Containers will still be sized as if the content was not moved.

## Example

<TypstPreview
  code={`
  #rect(inset: 0pt, fill: gray, move(
    dx: 4pt, dy: 6pt,
    rect(
      inset: 8pt,
      fill: white,
      stroke: black,
      [Abra cadabra]
    )
  ))
`}

## Accessibility

Moving is transparent to Assistive Technology (AT). Your content will be read in the order it appears in the source, regardless of any visual movement. If you need to hide content from AT altogether in PDF export, consider using [`pdf.artifact{:typst}`](/docs/reference/pdf/artifact/).

```typst
#move(
  dx: relative,
  dy: relative,
  body
) -> content
```
