## Text and Layout

- Added support for floating figures through the [`placement{:typst}`](/docs/reference/model/figure/#parameters-placement) argument on the figure function
- Added support for arbitrary floating content through the [`float{:typst}`](/docs/reference/layout/place/#parameters-float) argument on the place function
- Added support for loading `.sublime-syntax{:typst}` files as highlighting [syntaxes](/docs/reference/text/raw/#parameters-syntaxes) for raw blocks
- Added support for loading `.tmTheme{:typst}` files as highlighting [themes](/docs/reference/text/raw/#parameters-theme) for raw blocks
- Added _bounds_ option to [`top-edge{:typst}`](/docs/reference/text/text/#parameters-top-edge) and [`bottom-edge{:typst}`](/docs/reference/text/text/#parameters-bottom-edge) arguments of text function for tight bounding boxes
- Removed nonsensical top- and bottom-edge options, e.g. _ascender_ for the bottom edge **(Breaking change)**
- Added [`script{:typst}`](/docs/reference/text/text/#parameters-script) argument to text function
- Added [`alternative{:typst}`](/docs/reference/text/smartquote/#parameters-alternative) argument to smart quote function
- Added basic i18n for Japanese
- Added hyphenation support for `nb{:typst}` and `nn{:typst}` language codes in addition to `no{:typst}`
- Fixed positioning of [placed elements](/docs/reference/layout/place/) in containers
- Fixed overflowing containers due to optimized line breaks

## Export

- Greatly improved export of SVG images to PDF. Many thanks to [@LaurenzV](https://github.com/LaurenzV) for their work on this.
- Added support for the alpha channel of RGBA colors in PDF export
- Fixed a bug with PPI (pixels per inch) for PNG export

## Math

- Improved layout of primes (e.g. in `$a'_1${:typst}`)
- Improved display of multi-primes (e.g. in `$a''${:typst}`)
- Improved layout of [roots](/docs/reference/math/roots/#functions-root)
- Changed relations to show attachments as [limits](/docs/reference/math/attach/#functions-limits) by default (e.g. in `$a ->^x b${:typst}`)
- Large operators and delimiters are now always vertically centered
- [Boxes](/docs/reference/layout/box/) in equations now sit on the baseline instead of being vertically centered by default. Notably, this does not affect [blocks](/docs/reference/layout/block/) because they are not inline elements.
- Added support for [weak spacing](/docs/reference/layout/h/#parameters-weak)
- Added support for OpenType character variants
- Added support for customizing the [math class](/docs/reference/math/class/) of content
- Fixed spacing around `.{:typst}`, `\/{:typst}`, and `...{:typst}`
- Fixed spacing between closing delimiters and large operators
- Fixed a bug with math font weight selection
- Symbols and Operators **(Breaking changes)** 
  - Added `id{:typst}`, `im{:typst}`, and `tr{:typst}` text [operators](/docs/reference/math/op/)
  - Renamed `ident{:typst}` to `equiv{:typst}` with alias `eq.triple{:typst}` and removed `ident.strict{:typst}` in favor of `eq.quad{:typst}`
  - Renamed `ast.sq{:typst}` to `ast.square{:typst}` and `integral.sq{:typst}` to `integral.square{:typst}`
  - Renamed `.eqq{:typst}` modifier to `.equiv{:typst}` (and `.neqq{:typst}` to `.nequiv{:typst}`) for `tilde{:typst}`, `gt{:typst}`, `lt{:typst}`, `prec{:typst}`, and `succ{:typst}`
  - Added `emptyset{:typst}` as alias for `nothing{:typst}`
  - Added `lt.curly{:typst}` and `gt.curly{:typst}` as aliases for `prec{:typst}` and `succ{:typst}`
  - Added `aleph{:typst}`, `beth{:typst}`, and `gimmel{:typst}` as alias for `alef{:typst}`, `bet{:typst}`, and `gimel{:typst}`

## Scripting

- Fields 
  - Added `abs{:typst}` and `em{:typst}` field to [lengths](/docs/reference/layout/length/)
  - Added `ratio{:typst}` and `length{:typst}` field to [relative lengths](/docs/reference/layout/relative/)
  - Added `x{:typst}` and `y{:typst}` field to [2d alignments](/docs/reference/layout/align/#parameters-alignment)
  - Added `paint{:typst}`, `thickness{:typst}`, `cap{:typst}`, `join{:typst}`, `dash{:typst}`, and `miter-limit{:typst}` field to [strokes](/docs/reference/visualize/stroke/)
- Accessor and utility methods 
  - Added [`dedup{:typst}`](/docs/reference/foundations/array/#definitions-dedup) method to arrays
  - Added `pt{:typst}`, `mm{:typst}`, `cm{:typst}`, and `inches{:typst}` method to [lengths](/docs/reference/layout/length/)
  - Added `deg{:typst}` and `rad{:typst}` method to [angles](/docs/reference/layout/angle/)
  - Added `kind{:typst}`, `hex{:typst}`, `rgba{:typst}`, `cmyk{:typst}`, and `luma{:typst}` method to [colors](/docs/reference/visualize/color/)
  - Added `axis{:typst}`, `start{:typst}`, `end{:typst}`, and `inv{:typst}` method to [directions](/docs/reference/layout/stack/#parameters-dir)
  - Added `axis{:typst}` and `inv{:typst}` method to [alignments](/docs/reference/layout/align/#parameters-alignment)
  - Added `inv{:typst}` method to [2d alignments](/docs/reference/layout/align/#parameters-alignment)
  - Added `start{:typst}` argument to [`enumerate{:typst}`](/docs/reference/foundations/array/#definitions-enumerate) method on arrays
- Added [`color.mix{:typst}`](/docs/reference/visualize/color/#definitions-mix) function
- Added `mode{:typst}` and `scope{:typst}` arguments to [`eval{:typst}`](/docs/reference/foundations/eval/) function
- Added [`bytes{:typst}`](/docs/reference/foundations/bytes/) type for holding large byte buffers 
  - Added [`encoding{:typst}`](/docs/reference/data-loading/read/#parameters-encoding) argument to read function to read a file as bytes instead of a string
  - Added [`image.decode{:typst}`](/docs/reference/visualize/image/#definitions-decode) function for decoding an image directly from a string or bytes
  - Added [`bytes{:typst}`](/docs/reference/foundations/bytes/) function for converting a string or an array of integers to bytes
  - Added [`array{:typst}`](/docs/reference/foundations/array/) function for converting bytes to an array of integers
  - Added support for converting bytes to a string with the [`str{:typst}`](/docs/reference/foundations/str/) function

## Tooling and Diagnostics

- Added support for compiler warnings
- Added warning when compilation does not converge within five attempts due to intense use of introspection features
- Added warnings for empty emphasis (`__{:typst}` and `**{:typst}`)
- Improved error message for invalid field assignments
- Improved error message after single `#{:typst}`
- Improved error message when a keyword is used where an identifier is expected
- Fixed parameter autocompletion for functions that are in modules
- Import autocompletion now only shows the latest package version until a colon is typed
- Fixed autocompletion for dictionary key containing a space
- Fixed autocompletion for `for{:typst}` loops

## Command line interface

- Added `typst query{:typst}` subcommand to execute a [query](/docs/reference/introspection/query/#command-line-queries) on the command line
- The `--root{:typst}` and `--font-paths{:typst}` arguments cannot appear in front of the command anymore **(Breaking change)**
- Local and cached packages are now stored in directories of the form `\{namespace\}/\{name\}/\{version\}{:typst}` instead of `\{namespace\}/\{name\}-\{version\}{:typst}` **(Breaking change)**
- Now prioritizes explicitly given fonts (via `--font-paths{:typst}`) over system and embedded fonts when both exist
- Fixed `typst watch{:typst}` not working with some text editors
- Fixed displayed compilation time (now includes export)

## Miscellaneous Improvements

- Added [`bookmarked{:typst}`](/docs/reference/model/heading/#parameters-bookmarked) argument to heading to control whether a heading becomes part of the PDF outline
- Added [`caption-pos{:typst}`](/docs/reference/model/figure/#definitions-caption-position) argument to control the position of a figure's caption
- Added [`metadata{:typst}`](/docs/reference/introspection/metadata/) function for exposing an arbitrary value to the introspection system
- Fixed that a [`state{:typst}`](/docs/reference/introspection/state/) was identified by the pair `(key, init){:typst}` instead of just its `key{:typst}`
- Improved indent logic of [enumerations](/docs/reference/model/enum/). Instead of requiring at least as much indent as the end of the marker, they now require only one more space indent than the start of the marker. As a result, even long markers like `12.{:typst}` work with just 2 spaces of indent.
- Fixed bug with indent logic of [`raw{:typst}`](/docs/reference/text/raw/) blocks
- Fixed a parsing bug with dictionaries
